# Experience Cloud集成概述

Adobe Experience Cloud是一套全面的同类最佳集成应用程序和服务，它们构建在通用数据平台上，并具有一组通用的强大功能。

## 为 Platform 服务启用 Experience Cloud 应用程序 {#section_A3D024994DA3492F8435CFCC4EF035C2}

帮助：[为 Platform 服务启用应用程序](core-services/core-services.md#concept_07ED1D5C64234E77976E6D572E78FB9C)

介绍如何执行操作：

* 在 Experience Cloud 中配置您的公司。
* 让您成为管理员。
* [实施 Experience Cloud ID 服务](https://experienceleague.adobe.com/docs/id-service/using/home.html?lang=zh-cn)。
* 通过平台数据收集使您的[!DNL Analytics]和[!DNL Target]实施符合现代化要求。
* 开始使用核心服务.

解决方案或服务：

* 激活 — Experience Platform数据收集（以前称为Launch）
* Analytics
* Target
* [Experience Cloud ID 服务](https://experienceleague.adobe.com/docs/id-service/using/home.html?lang=en)

## Experience Cloud ID 服务 {#section_6ECCCFA2D84D4D4F88C879C799CA9D78}

ID 服务提供了一个通用的永久性 ID，用于在 Experience Cloud 的所有解决方案中标识您的访客。它可以替换各种服务(如Analytics、Audience Manager、Adobe Target、视频心率)以及其他Experience Cloud应用程序和产品的ID生成代码。

请参阅 [Experience Cloud ID 服务](https://experienceleague.adobe.com/docs/id-service/using/home.html?lang=en)

**适用的解决方案或服务**

* [Adobe Analytics](https://experienceleague.adobe.com/docs/id-service/using/home.htmlmcvid-setup-analytics.html?lang=en)
* [Adobe Target](https://experienceleague.adobe.com/docs/id-service/using/home.htmlmcvid-setup-target.html?lang=en)
* [[!UICONTROL Data Workbench]](https://experienceleague.adobe.com/docs/id-service/using/home.htmlmcvid-dwb.html?lang=en)

## 受众 {#section_5F60D7B0833348B9A1D74663AADCB42C}

帮助：[受众](audience-library/audience-library.md#topic_679810123CAA4E0CA4FA3417FB0100C7)

在 Experience Cloud Audience Library 中创建和管理受众。受众可从各种来源创建或派生，如：

* 在 [!DNL Experience Cloud] 中创建新受众。
* 从 [!DNL Analytics] 发布到 [!DNL Experience Cloud] 的区段。
* 从 [!DNL Audience Manager].

**适用的解决方案或服务**

* [Adobe Target 中的活动](https://experienceleague.adobe.com/docs/target/using/activities/activities.html?lang=en)
* Audience Manager 中的[分段](https://experienceleague.adobe.com/docs/analytics/integration/audience-analytics/audience-analytics-workflow/aam-analytics-segments.html?lang=en)
* [Media Optimizer](https://enterprise.efrontier.com/CMDashboard?ticket=JrciD7q2bF1y2mDWFHmEyibmxtHqnZFSOMml-n993zOBc-ovZGNZkX5vgePWqKNMoMmPSqf9PkzFeYF4UN6GqSXDVNDvwgnvv9KT8PvVxk8%3D)（需要登录）

## 客户属性 {#section_6A9EA6847F654F129381869E5016626C}

## 激活 {#section_A23510A2D57842F6BAD043650C06DE42}

帮助：[快速入门](https://experienceleague.adobe.com/docs/launch/using/get-started/quick-start.html?lang=en#get-started)

使用 Experience Cloud 激活解决方案来设置和调试您的 Experience Cloud 解决方案。

1. 使用 [Experience Platform Launch](https://experienceleague.adobe.com/docs/launch/using/overview.html?lang=en) 或 [Dynamic Tag Management](https://experienceleague.adobe.com/docs/dtm/using/dtm-home.html?lang=en) 插入可在您的页面上激活 [Adobe Experience Cloud 解决方案](solutions-core-services.md#topic_BD726D3A649E4FC49063029E86B70C62)的代码。
1. 使用 [Adobe Cloud Platform Auditor](https://experienceleague.adobe.com/docs/auditor/using/overview.html?lang=en) 对您的实施进行测试。

使用 Adobe Experience Cloud Debugger 扩展来调试 Auditor 发现的问题或检查有关您实施的其他信息。

**适用的解决方案或服务**

* [Analytics](https://experienceleague.adobe.com/docs/analytics/analyze/analysis-workspace/home.html?lang=en)
* [Audience Manager](https://experienceleague.adobe.com/docs/dtm/using/tools/audiencemgmt.html?lang=en)
* [Media Optimizer](https://experienceleague.adobe.com/docs/dtm/using/tools/media-optimizer.html?lang=en)
* [Adobe Target](https://experienceleague.adobe.com/docs/dtm/using/tools/target.html?lang=en)

## [!DNL Adobe Advertising] {#section_9B1935F8BBC147C89C6DB68A35CB1BAB}

帮助（需要登录）：[与 Adobe Experience Cloud 解决方案和服务的集成](https://enterprise.efrontier.com/CMDashboard/help/internal/concepts_and_features/media_optimizer_integration_with_adobe_marketing_cloud.htm)

**适用的解决方案或服务**

**Analytics:** 可以每天将网站参与度和转化数据发送 [!DNL Adobe Advertising]到，以便将这些数据用于广告优化和报告。此外，[!DNL Advertising]可以每天将搜索引擎和社交网络流量数据发送到Analytics，以便该数据可用于Reports &amp; Analytics、Report Builder和Ad Hoc Analysis功能中的报表。

**动态标签管理器：** 您可以使用 [动态标签管理器为您的搜索、社交和显示广告登陆页面创建基于广告像素的转化跟踪标签](https://docs.adobe.com/content/help/zh-Hans/dtm/using/tools/media-optimizer.html)，以及第三方跟踪标签。（您还可以直接在[!DNL Advertising]中创建[!DNL Advertising]标记。）

**Experience Cloud 受众：**（具有显示管理需求的广告商）您可以将任何 [Adobe Experience Cloud 受众](https://docs.adobe.com/content/help/zh-Hans/core-services/interface/audiences/audience-library.html)用作显示广告的目标。您可以自动使用在Adobe Experience Cloud中创建的受众和已发布到Adobe Experience Cloud的Analytics受众，并且在将[!DNL Adobe Advertising]帐户配置为允许后，您可以使用来自Audience Manager的受众。 有关访问Adobe Experience Cloud和“配置文件与受众”，以及有关[!DNL Adobe Advertising]和Adobe Experience Cloud受众之间初始设置的更多信息，请联系您的客户经理。 **注意：**&#x200B;如果您还使用 Adobe Target，则您发布到 Adobe Experience Cloud 的任何受众也可用于 Adobe Target 中的活动。

**Experience Cloud Assets：**（具有显示管理需求的广告商）您可以通过新的显示测试版视图，将任何 Adobe Experience Cloud 资产用作显示广告的创意。您必须[登录通过Adobe Experience CloudAdobe广告](https://enterprise-test.efrontier.com/CMDashboard/help/internal/getting_started/t_log_in_from_adobe_marketing_cloud.htm))才能访问Adobe Experience Cloud资产。 有关访问 Adobe Experience Cloud 的信息，请联系您的帐户管理员。

**Experience Cloud 通知：**&#x200B;通过每个页面顶部的通知链接，可以查看从测试版搜索警告模板生成的所有警告，以及 Adobe Experience Cloud 系统更新、帖子、提及次数和共享资产。您必须[登录通过Adobe Experience CloudAdobe广告](https://enterprise-test.efrontier.com/CMDashboard/help/internal/getting_started/t_log_in_from_adobe_marketing_cloud.htm))才能访问通知。 有关访问 Adobe Experience Cloud 的信息，请联系您的帐户管理员。
